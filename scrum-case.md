# Scrum案例分析
+ 产品（Product）
+ 产品待办项（Backlog Item）
+ 发布（Release）
+ 冲刺（Sprint）
+ 每个待办项都包含一些任务（Task）
+ 每个任务都拥有一个估算记录条目（Estimation Log Entry）集合
+ 发布中包含计划好的待办项（Scheduled Backlog Item）
+ 冲刺中包含已提交的待办项（Committed Backlog Item）
+ 租户（Tenant）
+ 用户（User）
+ 权限（Permission）
+ 讨论（Discussion）
+ 论坛（Forum）
+ 讨论帖（Post）
+ 共享日历（Shared Calendar）
+ 支付（Payment）
+ 销售支持计划（Support Plan）
+ 跟踪支持事件（Incident）
+ 支持（Support）
+ 账户（Account）
+ 团队（Team）
+ 产品负责人（Product Owner）
+ 团队成员（Team Member）
+ 人力资源利用率（Human Resource Utilization）
+ 利用率（Utilization）
+ 可用性（Availability）
+ 日常的日历条目（Calendar Entry）
+ 提醒（Reminder）
+ 团队里程碑（TeamMilestone）
+ 计划会议（Planning Meeting）
+ 回顾会议（Retrospective Meeting）
